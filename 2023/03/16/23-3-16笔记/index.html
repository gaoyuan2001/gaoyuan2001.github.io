Springboot

2023/3/16 学习 Java

Springboot：

1、减少xml配置，遵循约定大于配置（也可以配置，但尽量使用约定），简化配置

2、使用大量注解，完成配置任务。

注解与xml的优缺点

  • 注解：hard code，硬编码，修改还维护麻烦
  • 使用注解配置，会分散到不同的类上，管理维护麻烦
  • 注解和类的职能混在一起，伴随着大量注解
  • 简单，快捷，开发效率高
  • 流行微服务，使用springboot配置注解非常快

3、还可以使用配置类，非常灵活

默认支持SpringMVC + spring

spring boot starter启动器：在应用启动时，自动完成某个框架的默认配置，校验

默认约定主启动类包下的所有的子包，将会自动扫描（主启动类：SpringbootApplication）

微服务RESTful请求

restful不是一个规范，而是一个建议

将网络上的一切东西都视为资源。将动作理解成服务类型的资源。使用名词来表示

路径建议使用浮数

请求类型：

  • GET：用于获取资源

  • POST：用于新增资源

  • PUT：完整修改资源

  • PATCH：部分修改资源，打补丁，修改

  • DELETE：删除资源

  • OPTION：获取一些指引，操作向导

状态码：

  • 2xx：200，一切ok，请求成功
  • 3xx：重定向方面的问题
  • 4xx：请求出现的问题
  • 5xx：服务器端出现的错误

微服务：

返回值类型：ResponseEntity(Map< String, object>)

可以返回状态码

#允许浏览器或者某种前打算以_method向后台提交请求类型
spring.mvc.hiddenmethod.filter.enabled=true
#可以通过以类型的请求向后台传递参数
spring.mvc.formcontent.filter.enabled=true
