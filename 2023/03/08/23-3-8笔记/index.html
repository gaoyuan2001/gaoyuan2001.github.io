SpringMVC

搭建springmvc框架的过程

第一步：引入springmvc库

pom.xml

<!-- https://mvnrepository.com/artifact/org.springframework/spring-webmvc -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-webmvc</artifactId>
    <version>5.3.25</version>
</dependency>

第二步：在web.xml中配置mvc

<!--配置springmvc-->
<!-- 一、配置中文编码过滤器，统一utf-8编码 -->
<filter>
   <filter-name>CharacterEncodingFilter</filter-name>
   <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
   <init-param>
       <param-name>encoding</param-name>
       <param-value>UTF-8</param-value>
   </init-param>
   <init-param>
       <param-name>forceEncoding</param-name>
       <param-value>true</param-value>
   </init-param>
</filter>
<filter-mapping>
   <filter-name>CharacterEncodingFilter</filter-name>
   <url-pattern>/*</url-pattern>
</filter-mapping>
<!-- 二、springmvc配置，注意：url-pattern可以配置成*.do或者*.action或者/，但是不能配置成/* -->
<!-- /*：可以匹配所有请求，但不能匹配带扩展名的请求，比如*.do，*.action了 -->
<!-- *.action：匹配所有以action结尾的请求 -->
<!-- /：可以匹配所有请求，包括静态资源请求 -->
<servlet>
   <servlet-name>springmvc</servlet-name>
   <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
   <!-- 配置springmvc的配置文件路径 -->
   <init-param>
       <param-name>contextConfigLocation</param-name>
       <param-value>classpath:springmvc.xml</param-value>
   </init-param>
   <!--应用启动时加载-->
   <load-on-startup>1</load-on-startup>
</servlet>
<servlet-mapping>
   <servlet-name>springmvc</servlet-name>
   <url-pattern>/</url-pattern>
</servlet-mapping>
DispatcherServlet：负责分发请求

第三步：配置springmvc.xml文件

  • 扫描包的基路径：com.yuan.chapter1

    从其中扫描出来Controller组件的注解，将所有的放在一个容器内

  • 解析静态资源

    虽然已经也接收静态资源，但是转发给tomcat默认的servlet处理

  • 配置静态资源映射

    location：目录

    mapping：要找的东西

    也就是如果找mapping的静态资源，自动去到location下去找

  • 视图层

    自动增加前缀和后缀，组成存储转发的路径

  • 注解驱动

<!-- 一、配置基础扫描包路径，springmvc会自动扫描该包及其子包下满足特定条件的类，use-default-filters属性默认值为true，表示使用默认过滤器，默认过滤器会扫描所有标有@Component、@Controller、@Service、@Repository注解的类 -->
<context:component-scan base-package="com.situ.springmvc1/>
<!-- 二、解析静态资源，需要与结合mvc:annotation-driven -->
<mvc:default-servlet-handle/>
<!-- 三、配置静态资源映射，不仅可以将任意目录下的静态资源映射成指定的路径，还能设置浏览器对此静态资源缓存 -->
<mvc:resources location="/assets/" mapping="/static/**/>
<!-- 四、视图层配置，使用jsp作为模板引擎 -->
<bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <property name="prefix" value="/WEB-INF/jsp/"/>
    <property name="suffix" value=".jsp"/>
</bean>
<!-- 五、注解驱动：激活基于注解的配置，如：@RequestMapping、@ExceptionHandler、数据绑定、@NumberFormat、@DateTimeFormat、@Controller、@Valid、@ResponseBody等 -->
<mvc:annotation-driven/>

第四步：编写Controller组件

DispatcherServlet：把响应的请求转发给响应的控制器组件

加注解@Controller，代表是一个控制器

@RequestMapping(“ “)，不管是post还是get请求都可以处理，和servlet中的service差不多

@GetMapping(“ “)，只能接收get请求

@PostMapping(“ “)，只能接收post请求

常用的注解及其相关概念操作：

1、@Controller，代表控制器组件

@RestController，代表响应json数据的控制器组件

2、@RequestMapping：（请求映射）表示接收类型的模式，可以修饰类，也可以修饰方法

代表各种类型的请求，，例如定义为“/student”，则/student/list请求，就会进入此类，和servlet中的@WebServlet(“/student/*”)一样

修饰类的时候，表示类中所有的请求，以此为基础

处理的请求可以写多个，写成数组，可以定义多个

@RequestMapping({"/hello", "/stu"})

3、@GetMapping：仅仅是get请求，仅可以修饰方法

@PostMapping：仅仅是post请求，仅可以修饰方法

@GetMapping("/list")

4、在controller的方法中，定义Map、Model、ModelMap，使用此三种类型的对象，都可以等效于对请求域的操作

Map用put（常用)

ModelMap用addAttribute、或者put（此类继承自LinkedHashMap）

5、存储转发和重定向。返回字符串的三种含义

存储转发：服务器转发，客户端无感知，地址无变化，不创建新的请求

重定向：客户端转发，客户端知道，地址已发生变换，创建新的请求

返回值：加前缀forward，表示存储转发给另一个请求

返回值：加前缀redirect，表示重定向给另一个请求

关键字：forward，存储转发给一个请求

return ”forward:/student/list";//表示转发给一个/student/list请求

HttpServletRequest req

定义一个请求获取上下文

req.getContextpath()

6、参数绑定（获取参数），自动将请求中的参数绑定到方法中的参数

  • 如果名称相同，直接绑定

  • 如果类型不同，自动转换类型（有可能有异常）。

    基本数据类型的封装类型，String，Date，LocalDate/LocalTime/LocalDateTime，BigInteger，BigDecimal

  • 日期类型：加注解@DateTimeFormat(pattern = “yyyy-MM-dd”)

  • 文件类型：MultipartFile，用于文件上传

  • 使用@RequestParam注解，可以指定请求参数，使用不同名称绑定，并可以指定此参数是否必需

@RequestParam(name = "pageNo", required = true) Integer pn
  • 如果使用@RequestParam修饰map，此时map将不作为域对象，而作为存储请求参数的map
  • 可以直接对javabean（模型类）进行绑定，但是需要setter/getter方法

7、响应ajax请求

  • 指定响应内容类型：produces = “application/json;charset=utf-8”
  • 添加@ResponseBody注解
@GetMapping(value = "/add", produces = "application/json;charset=utf-8")
    @ResponseBody
    public Student save() {
        Student stu = new Student();
        stu.setId(100);
        stu.setName("孙小美");
        stu.setSex("女");
        return stu;
    }
  • 引入依赖jackson，用于json序列化（把json变成字符串）
<!--用于json序列化-->
<!-- https://mvnrepository.com/artifact/com.fasterxml.jackson.core/jackson-databind -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.14.2</version>
</dependency>
<!-- https://mvnrepository.com/artifact/com.fasterxml.jackson.core/jackson-core -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-core</artifactId>
    <version>2.14.2</version>
</dependency>
<!-- https://mvnrepository.com/artifact/com.fasterxml.jackson.core/jackson-annotations -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-annotations</artifactId>
    <version>2.14.2</version>
</dependency>

controller中方法的参数可以是（任意顺序进行组合）：

1、基本数据类型封装类性，String，LocalDate/LocalTime/LocalDateTime，Date，BigeInteger，BigDecimal

2、javaBean

3、Map、Model、ModelMap，等效于请求域对象使用

4、HttpServletRequest，HttpServletResponse，HttpSession（不建议使用）

5、MultipartFile：用于文件上传

controller中方法的返回值可以是：

1、字符串。三种定义

2、可以返回javabean，或者集合，挥着字符串：当方法响应ajax请求时，返回值会被序列化成json对象

3、ModelAndView：同时作为域对象存储模型数据，以及存储视图名称

  • setViewName()，返回视图，jsp
  • addObject()，返回数据，data
