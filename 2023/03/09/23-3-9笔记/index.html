SpringMVC拦截器

拦截器与过滤器的区别？

  • 拦截的时机（位置）不同，过滤器是在到达servlet之前拦截，拦截器是在到达controller之前拦截

  • 过滤器是javaee内置的组件，而拦截器是springmvc提供的

  • 拦截器可以使用springmvc的一些特性，但过滤器不行

步骤

1、配置拦截器

在springmvc .xml中配置

<!--拦截器的配置-->
<mvc:interceptors>
    <!--配置一个拦截器-->
    <mvc:interceptor>
    <!--配置拦截模式，定义多个。/*,*.do,/**/*-->
    <!--/**代表任意多层目录-->
    <mvc:mapping path="/**"/>
    <!--排除拦截的请求-->
    <mvc:exclude-mapping path="/user/login"/>
    <mvc:exclude-mapping path="/user/logout"/>
    <mvc:exclude-mapping path="/**/*.js"/>
    <mvc:exclude-mapping path="/**/*.png"/>
    <mvc:exclude-mapping path="/**/*.jpg"/>
    <mvc:exclude-mapping path="/**/*.css"/>
    <mvc:exclude-mapping path="/static/**"/>
    <bean class="com.situ.springmvc1.interceptors.SecurityInterceptor"/>
    </mvc:interceptor>
    <!--继续配置拦截器-->
</mvc:interceptors>

/**：代表任意多层目录请求和

/*：代表一层目录请求

2、定义拦截器：

实现接口：HandlerInterceptor

返回true代表放行，返回false代表拦截

public class SecurityInterceptor implements HandlerInterceptor {
    //到达controller之前处理，返回true代表放行，返回false代表拦截
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse
    response, Object handler) throws Exception {
    System.out.println("到达controller之前");
    return true;
    }
    //到达controller之后处理
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse
    response, Object handler, ModelAndView modelAndView) throws Exception {
    System.out.println("到达controller之后");
    }
}

上传文件

  • 在springmvc配置上传文件解析器（多段文件内容解析器），multipartResolver
<!--配置文件上传-->
    <!-- 文件上传配置。注意：文件上传时，要配置form的enctype属性值为multipart/form-data -->
    <!-- 文件上传解析器，其id固定为multipartResolver，配置为其它值无效，此实现类依赖于
commons-fileupload、commons-io -->
    <bean id="multipartResolver"    class="org.springframework.web.multipart.commons.CommonsMultipartResolver">
    <!-- 上传文件大小上限，单位为字节（10MB） -->
    <property name="maxUploadSize" value="10485760" />
    <!-- 请求的编码格式，必须模板页的编码一致，以便正确读取表单的内容，默认为ISO-8859-1-->
    <property name="defaultEncoding" value="UTF-8" />
    </bean>
  • 配置文件上传依赖（pom），Commons-FileUpload
<!--配置文件上传-->
<!-- https://mvnrepository.com/artifact/commons-fileupload/commons-fileupload -->
<dependency>
    <groupId>commons-fileupload</groupId>
    <artifactId>commons-fileupload</artifactId>
    <version>1.5</version>
</dependency>

Spring

###控制反转，IOC：把创建对象的权力，交给框架，由框架帮你创建（框架同时可以帮助约束对象，规范）

依赖注入，DI：框架可以帮助自动将某对象依赖的其他对象进行注入（如，service依赖dao，自动将dao注入到service）

StudentServiceImpl ssi = new AnotherStudentServiceImpl();

StudentDAO dao = new StudentDAOImpl();

ssi.setStudentDAO(dao);

正常情况下，不使用框架：

问题1：大量创建对象。但实际上并不需要

问题2：在运行期间，很难切换业务的实现，每次都需要重新启动，不灵活

Spring可以用来整合其他框架

使用Spring框架步骤

1、引入依赖。如果web程序中已经引入SpringMVC，则Spring相关库会被依赖引入，无需再单独引入

2、在web.xml中，配置spring监听器

<!--配置spring监听器-->
<!-- spring配置文件参数 -->
<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:spring.xml</param-value>
</context-param>
<!-- spring引入，在web应用启动时执行-->
<listener>
    <listener·class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>

注入有三种（spring.xml）：

在spring.xml中设置

1、字段注入

2、方法注入

3、构造函数注入

以上都是老旧方法：

控制反转
<bean id="studentService"class="com.example.demo.service.impl.StudentServiceImpl">
    构造函数注入
    <constructor-arg index="0" ref="studentDAO" />

    字段注入，方法注入
    <property name="dao" ref="studentDAO"  />
</bean>

<!--控制反转-->
<bean id="studentDAO" class="com.example.demo.service.dao.impl.StudentDAOImpl" />
4、通过扫描基础包，结合注解实现自动注入

注解：@Controller @Service @Repository @Component

依赖注入：在构造函数上加注解@Autowired，按类型匹配注入，需要容器中有唯一的指定类型的实例。

@Autowired 表示依赖其他bean。按类型匹配注入。

目前常用方法：

<!--通过设置扫描基包，结合注解实现自动注入-->
<!--@Controller @Service @Repository @Component-->
<context:component-scan base-package="com.yuan" />
