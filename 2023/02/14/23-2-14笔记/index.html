红黑树：一种自平衡二叉查找树

红黑树时一种含有红黑节点并能自平衡的二叉查找树

hashmap底层实现是 数组 + 链表 —> 数组 + 红黑树

链表长度大于8时转换为红黑树

红黑树不是严格意义上的平衡二叉树

结点需要标注颜色：color：black、red

  • 1.根节点是黑色的，每个叶子节点是黑色

  • 2.每个红色结点的两个子结点一定都是黑色

  • 3.任意一结点到每个叶子结点的路径都包含数量相同的黑结点

  • 4.如果一个结点存在黑子结点，那么该结点肯定有两个子结点

B树（B-树）

多路查找：深度浅，IO次数少

B-树就是B树(Balanced Tree)，是一种平衡的多路查找树。一棵m阶B树(Balanced Tree of order m)是一棵平衡的m路搜索树。它或者是空树，或者是满足下列性质的树：

  • 树中的每个结点至多有m棵子树。

  • 若根结点不是叶子结点，则至少有两棵子树。

  • 除根之外的所有非终端结点至少有[m/2]棵子树。

  • 所有的非终端结点中包括如下信息的数据

    （n,A0,K1,A1,K2,A2,….,Kn,An）

    其中：Ki（i=1,2,…,n）为关键码，且Ki < K(i+1)，

    Ai 为指向子树根结点的指针(i=0,1,…,n)，且指针A(i-1) 所指子树中所有结点的关键码均小于Ki (i=1,2,…,n)，An 所指子树中所有结点的关键码均大于Kn.

    n 为关键码的个数。

  • 所有的叶子结点都出现在同一层次上，并且不带信息（可以看作是外部结点或查找失败的结点，实际上这些结点不存在，指向这些结点的指针为空）。

B+树：二叉排序树 + 平衡二叉树 + B树

数据仅存储于叶子结点上

B树和B+树的应用场景：数据库索引，大数据

B+树是B-树的变体，也是一种多路搜索树：

其定义基本与B-树同，除了：

  • 1.非叶子结点的子树指针与关键字个数相同；

  • 2.非叶子结点的子树指针P[i]，指向关键字值属于[K[i], K[i+1])的子树（B-树是开区间）；

  • 3.为所有叶子结点增加一个链指针；

  • 4.所有关键字都在叶子结点出现；

注解：就是注释的一种

其一类型的注解，只有标识作用，在编译后即擦除，比如：@Override，@FunctionalInterface

还有一类注解，结合反射，可以起到一定的程序逻辑，编译后依然存在

自定义注解：

1.使用@interface，不具备继承能力

2.类似于定义抽象方法，但是无参，可以通过default定义默认值。抽象方法返回值的类型就是值的类型

3.{}代表数组

4.返回值只能是基本数据类型（封装类），数组，或者引用类型，不能使用集合

关键字：

@Target(Element.Type)//此注释将放在哪里

@Retention(RententionPolicy.RUNTIME)此注释的作用时间，SOURCE（源码时期）、ClASS（类时期）、RUNTIME（运行时期）

结合反射，可以起到一定的程序逻辑，编译后依然存在

必须结合反射才能拿到注解里的值

Class<?> clazz = A.class;

Description annotation = clazz.getDeclareAnnotation(Description.class);

System.out.println(Arrays.toString(annotation.author);
System.out.println(annotation.version);

  • 1.函数式接口。

    只能有一个抽象方法（不包含toString()，equals() 等Object中的方法）的接口，可以添加@FunctionalInterface接口，但不是必须的

    Runnable就是一个函数式接口

    @FunctionalInterface
public interface Study {
    public void study(Study a);//一个抽象方法
    
    String toString();//Object类里的方法
    
    public static void sayHello(){//静态方法
        System.out.println("Hello");
    }
    
    public default void play(){//默认方法
        System.out.println("play");
    }
}

main{
    Study.super.play();//实现类（或者实现类的子类）调用接口里的play默认方法，接口名.super.默认方法名
}

  • 2.接口的静态方法和默认方法，

    静态方法：static修饰。接口.方法名调用

    默认方法：default修饰。直接被继承到其实现类中，也可以在实现类重写，但对实现类及其子类不会产生任何影响，

    　　　　　实现类调用：接口名.super.默认方法名（自动默认全有）

  • 3.lambda表达式与方法引用，（只能对函数式接口使用）

    括号里的是接口的抽象方法的参数列表，-> 箭头后面表示方法体，前提A必须是函数式接口

    当参数个数只有一个时，可以省略小括号，也可以省略数据类型

    当参数个数有多个时，可以省略数据类型

    如果方法体只有一句可以省略大括号，也可以省略return

    如果方法体有多句时，不可以省略大括号

    A a = () -> System.out.println("执行任务")；

study((int a)-> System.out.println(a + "执行任务"));//接口的抽象方法

    //线程中lambda表达式的应用
//这里也是实现了接口函数式接口Runnable，其中的唯一抽象方法run()
//所以这里可以用lambda表达式代替run()方法
//原来写成：
Thread t = new Thread(new Runnable() {
            @Override
            public void run() {
                System.out.println("aaaa");
            }
        });
        t.start();
//用lambda表达式更改后：
Thread t = new Thread(()-> System.out.println("aaaa"));
t.start();


//自己创建函数式接口
public interface Logger {
    void info(String msg);
}

//这里的()对应的是函数式接口的抽象方法的参数列表，因为定义Logger接口时有唯一的带参的抽象函数，所以要有一个t
//t的类型可以省略，到时候调用的时候自动匹配
Logger logger = (t) -> System.out.println(t);

//这里用的时方法引用::，你的想法是输出一个参数，而System.out.println()正好是把括号内的一个参数输出，返回值都为空，所以你可以直接引用此方法为你服务
Logger logger2 = System.out::println;

    方法引用::，双冒号

    当你想执行的方法，和一个已有的方法参数一样时（返回值，参数列表，方法体），你可以使用方法引用直接引用

    study(Integer :: sum);

  • 4.Stream流操作

    中间操作：返回的值仍然是流，还可以继续操作

    终结操作：返回的值不是流类，可以直接输出了

    List<Integer> list = List.of(1, 2, 3, 4, 5);
//列表的流，list.stream()
//filter()过滤方法，把满足条件的元素筛选出来
list.stream().filter(t -> t % 2 == 0)
        .forEach(System.out::println);//输出：2，换行，4

//此语句是对上面的改版，采用并行的方法同时筛选，同时输出的结果
//列表的并行流，多线程的运用
String s = list.parallelStream()
        .filter(t -> t % 2 == 0)
        .map(t->String.valueOf(t))//map方法是映射，把t类型转换为String类型，并返回String类型
        .collect(Collectors.joining(","));//把并行的结果用，连接起来

System.out.println(s);//输出：2，4

//以下是整数流
//此算法是计算出1到100的叠加
//rangeClose(1,100)是取闭区间[1，100]
//redece(,),其参数列表正好是两个，返回值是一个，正好和我们需求一致，所以可以直接使用lambda表达式
OptionalInt oi =  IntStream.rangeClosed(1, 100).reduce((a, b) -> a + b);

System.out.println(oi.getAsInt());//输出：5050
