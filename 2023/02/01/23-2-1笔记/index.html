多线程

进程和线程的区别：process、thread

进程：系统分配资源和调用的最基本单位，每个程序可能不止一个进程

线程：CPU调用分派的最小单位，每个进程最少包含一个线程（主线程），在CPU上运行的其实是线程

系统把资源（内存，CPU）分配给进程，此进程的所有线程共享这些资源，就会造成线程冲突

串行：1——2——3——

并行：1——

　　　2——

　　　3——

并发：1—　　　　1—

　　　　　2—　　　　2—

　　　　　　　3—　　　　3—

并行：同时处理多个任务的能力

并发：处理多个任务的能力

一个线程的run() {……}，就是这个线程的任务

生命周期：当run里的代码执行完后，此线程就结束了，死亡了

创建—start—>就绪态—run—>运行——>结束(死亡)

start()：启动线程，到就绪态，CPU可以调用线程

run()：执行线程，线程获得时间片timeslice，在CPU上运行

创建线程的方法：

一、创建Thread的子类，重写run方法，启动

二、创建Runnable的接口的实现类，作为Thread构造器的参数

三、jdk1.5后新增Executor，Executors，ExecutorService

Executor（执行器）：接口，用于提交任务（多线程）

Executors：类，提供一些快速创建Executor的工具类

ExecutorService：接口，继承自Executor，并扩展了一些接口

ExecutorService es = Executors. newFixedThreadPool(5);
es.execute(r);// r是继承自Runnable接口的实现类的对象，里面重写了run方法

Thread.yield(); // 向CPU提出建议，去执行其他线程，可以放在run里面

Runnable：没有返回值

Callable：可以有返回值

Future类：代表一个未来有值的对象

Thread.sleep()：休眠指定的毫秒数，需要捕获异常

  • execute只能提交Runnable类型的任务，无返回值。

    submit既可以提交Runnable类型的任务，也可以提交Callable类型的任务，会有一个类型为Future的返回值，但当任务类型为Runnable时，返回值为null。

  • execute在执行任务时，如果遇到异常会直接抛出，而submit不会直接抛出，只有在使用Future的get方法获取返回值时，才会抛出异常。

Callable<Integer> c = new Callable<>() {
    public Integer call() throws Exception {
        Thread.sleep(3000);
        return 103;
    }
};
//Future：代表一个未来有值的对象
Future<Integer> f = es.submit(c);// f可能还没有计算出来，但是先执行之后的语句，相当于，同时执行

Thread。sleep(8000);

try{
    Integer i = f.get();// 此时的f应该计算出来了，节约时间
    System.out.println(i);
} catch(……) {
    
} catch(……) {
    
}

仅在面试中有用：

wait()：表示等待，可以指定具体的等待时间，或者无限等待。

　　　　wait();用锁来调用，唤醒也需要同一把锁来唤醒

　　　　wait(3000);也可以等待三秒后，自动唤醒，无需手动notify唤醒

private static final byte[] lock = new byte[0];
synchronized(lock) {
    lock.wait();
    System.out.println("等待中");
}
synchronized(lock) {
    lock.wait(3000);// 无需手动唤醒
    System.out.println("等待中");
}

notify()：表示唤醒一个等待中的线程，这个锁如果有多个无参等待，则唤醒只会随机唤醒其中一个，

notifyAll()：表示唤醒所有等待中的线程

等待和唤醒必须放在同步方法，或者同步块中

synchronized(lock) {
    lock.notify();
    System.out.println("已唤醒");
}

synchronized：同步锁，必须为对象，例如引用对象synchronized(lock) {……}

　　　　　　　　也可以定义在方法上，给方法加锁public synchronized void ab() {……}

锁池：没有拿到锁的线程，竞争关系，线程等待拿到锁，转移出锁池，才能执行

等待池：等待中的线程，唤醒（notify）后的线程会被转移到锁池

反射：Java的灵魂技术。Reflect

一种根据类的原数据，创建对象，访问对象成员的技术

专门用于描述其他类的类，叫Class。

如何得到Class类型的 对象 ？不是new出来的，是获取来的

  • 1.类名.class;

    Class<?> clazz1 = Student.class;

  • 2.通过对象.getClass();

    Student s = new Student();

    Class<?>clazz2 = s.getClass();

    局限性：如果没有对象，则获取不到

  • 3.通过Class.forName();需要捕获异常

    Class<?>clazz3 = Class.forName(“com.yuan.chapter1.Student”);

    灵活性高，括号里的变量可以改变，因此可以传参，用变量来获取

反射可以做什么？

1.创建对象：其实都是调用了之前类的构造函数，**因此 参数列表要统一**。

建议使用第二种构造函数创造对象

  • 用clazz方法，newInstance();

    Student stu = (Student)clazz.newInstance(String.class, String.class, int.class);// 也需要捕获异常

  • 用Constructor获取类的构造函数

    Constructor fun = (Constructor)clazz.getDeclaredConstructor(……);

    Constructor fun = (Constructor)clazz.getDeclaredConstructor(String.class, String.class, int.class);

2.可以访问（读写）对象的属性// Test4
Field

Field f = clazz.getDeclareField(“name”);// 括号里的使属性名

f.setAccessible(true);强行把private类型转为public

f.get( );

f.set( , );

3.可以访问对象的方法// Test5
Method

Method m = clazz.getDeclareMethod(“getName”);// 括号里是方法名

无法获取方法体。

使用m这个函数

m.invoke(null,”孙小美”,”女”,20);//null的位置填的是对象名，静态方法无需用对象调用，之后的是原函数的要用的参数

m.setAccessible(true);强行把private类型转为public

4.Method：类，描述方法的对象

5.Parameter:类，描述方法参数的队形

package com.yuan.chapter;
public class Test1 {
    public static void main(String[] args) {
        // 1.通过类名.class来得到Class对象
        // 需要知道类名
        Class<?> clazz1 = Student.class;

        // 2.通过原类的对象，对象.getClass()
        // 局限性：需要现有原类的对象，才能用此方法
        Student s = new Student();
        Class<?> clazz2 = s.getClass();
        /*
         * 3.通过Class.forName()来获取 
         * 但是需要注意两点： 
         * ①括号内的类型是String类型 
         * ②需要对其进行异常处理（try catch）
         * 
         * 此方法比以上两种方法更灵活，括号里的参数可以通过传参来获得
         */
        try {
            Class<?> clazz3 = Class.forName("com.yuan.chapter.Student");
            System.out.println(clazz1 == clazz3);// true
        } catch (ClassNotFoundException e) {
            // TODO Auto-generated catch block
            e.printStackTrace();
        }
        System.out.println(clazz1 == clazz2);// true
        System.out.println(clazz1.toString());//返回值为string类型 class com.yuan.chapter.Student
        System.out.println(clazz1.getName());//返回值为string类型 com.yuan.chapter.Student
        System.out.println(clazz1.getTypeName());//返回值为string类型 com.yuan.chapter.Student
        System.out.println(clazz1.getSimpleName());//返回值为string类型 Student	
    }
}

package com.yuan.chapter;
import java.lang.reflect.Field;
import java.lang.reflect.Modifier;
public class Test4 {

    public static void main(String[] args) {
        try {
            Class<?> clazz = Class.forName("com.yuan.chapter.Student");
            Student s = new Student();
            Field f = clazz.getDeclaredField("name");
            f.setAccessible(true);// 强制访问私有的属性
            f.set(s,"红姐");// 因为name在Student里是私有属性，所以需要写上一句
            System.out.println(f.get(s));// 红姐
            System.out.println(f.getName());// name，属性的名字
            System.out.println(f.getType().getName());// java.lang.String，属性的类的全名
            System.out.println(f.getType().getSimpleName());// String，属性的类的简单名
            System.out.println(Modifier.toString(f.getModifiers()));// private，属性的访问修饰符
                        
        } catch (Exception e) {
            e.printStackTrace();
        } 		
    }
}
