在关系数据库中，数据是存储在表中的。二维表

数据库的三范式：

1.第一范式：原子性：字段不可分。

表的每一列不可再分

2.第二范式：唯一性：有主键，非主键字段依赖主键。

满足第一范式的基础上，每一列都唯一和全部主键依赖。每个表只干一件事，只存储一种数据类型，必须和主键有关系

3.第三范式：冗余性：非主键不能相互依赖。

满足第二范式的基础上，某一列不可以通过其他列计算或推到而得出，比如年龄可以通过生日计算得出

在一定情况下也会使用反范式。

主键：primary key

联合主键：两列联合为为主键

逻辑主键：和业务没有关系，比如序号和本身并无对应联系

业务主键：和业务有关系，比如一个属性：学号，就和学生本身有关

外键：foreign key

开发中一般不会用到

外键：

cascade级联删除：会都删除

noaction：无行为

set null：设置空

设计数据库的表：约束

1.实体完整性约束：

  • 主键约束：每个设置主键
  • 自增长列：列自动增长
  • 唯一键约束：mysql通过索引来实现唯一键约束
  • 外键约束：保证主外键约束，保证数据完整性，在开发中几乎不适用外键约束

2.域完整性约束：对列的约束

  • 数据类型约束：每个列的类型不一样

    • tinyint，smallint，mediemint，int，bigint：整数
    • float，double：小数
    • char：定长字符串，varchar：不定长字符串
    • date：日期
    • time：时间
    • datetime：时间和日期
    • timestamp：时间戳，返回值是一个整数，距离1970年的毫秒数
    • bit：位，返回值是boolean类型
    • text：大文本类型
    • blob：大二进制类型
    • enum：枚举，可以提前设置范围，只能从中选，比如男，女 ，只能从两个中选
    • decimal：设置小数精确到几位，但是需要设置总长度
    • json：用于描述一个对象的一小段代码

  • 非空

  • 默认值约束
