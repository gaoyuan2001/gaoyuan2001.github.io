Java Web

2023/2/28 学习 Java

maven：构建工具，依赖管理工具

官方仓库：https://mvnrepository.com

Servlet：专业名词。表示执行web服务器中的java代码的组件

接收http请求，并进行响应

get/post请求的区别？

请求的类型：get/post

只有在表单中通过method属性明确指定为post的，才是post类型，除此之外，一切请求都是get请求

使用注解也可以配置servlet。

@WebServlet({“……”})

servlet生命周期

1、从第一次访问对应的请求，加载并创建对象。调用init()函数，共有两个init函数

servlet创建后会复用

可以通过参数指定servlet，从应用启动即加载。1

2、持续对外提供服务，等待处理请求

3、服务器（tomcat）终止应用，调用destroy函数

JSP：Java Server Page，Java服务器端页面

前端（浏览器）不能直接识别jsp，Tomcat可以识别，Tomcat可以把jsp转换成前端页面，再交给浏览器解析

jsp本质就是servlet。服务器（Tomcat），也可以称之为servlet容器，可以将jsp转译成servlet代码，并执行编译

默认访问webapp下的jsp

JSP四大作用域和九大内置对象（笔试）：

四大作用域

  • 页面域：4

    pageContext，当前页面

  • 请求域，每次请求都会创建一个请求对象、响应对象，一次请求结束两个同时销毁，1

    HttpServletRequest request：

    request.setAttribute(“currnt_login_user”,”张三”);

    request.getAttribute(“currnt_login_user”);

    仅限于一次请求

  • 会话域，生命周期长于请求域，2

    HttpSession session：

    仅限于一次对话

  • 应用域，生命周期更长，在整个应用程序存在期间有效。唯一一个，3

    ServletContext ctx = request.getServletContext();

    随应用启动而启动，随应用销毁而雄安会，整个应用唯一一个

九大内置对象，都是内置对象，可以在jsp直接用

1、request：代表请求对象，请求域

2、response：代表响应对象

3、out：字符输出流，直接写到前端界面

4、session：代表会话对象，会话域

5、application：应用域，类型为ServletContext

6、config：类型ServletConfig，servlet的配置对象

7、exception：异常对象。只有Error配置才有

8、page：指代当前页面，即this

9、pageContext：页面上下文，又称页面域

存储转发和重定向：重点

两者不能同时用

  • 重定向：把请求重新定向给另外一个新的请求，会产生一个新的请求

    请求调用

  • 存储转发：将当前请求作用域的数据转发给新的地址，但并不创建新的请求

    request.getRequestDispathcher(“……”).forward(request,response)

    响应调用
