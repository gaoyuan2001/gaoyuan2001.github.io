JavaSE基础语法

2023/1/4 学习 Java

多态

使用父类（祖先类）指向子类（后代类）的对象，此时访问引用的方法，实际上是子类的方法，有可能会表现出不同于父类的行为（方法重写）
这个类有很多子类，如果用这个类定义一个对象，则其对象可以的类型可以有很多种！！！

Teacher extends Person
Person tea = new Teacher();
父类的引用 指向了 子类的对象，
父类对象(Person)是真正的对象，只能调用父类已有的方法
tea是父类的对象，其内核是子类。

注意！！！
调用方法只能是调用父类的方法：
1.未经过子类重写的，调用父类的方法
2.经过子类重写的，调用子类的方法

抽象类

使用abstract关键字
抽象类不能实例化，也就是不能创建对象
当觉得某个类不应该被实例化，仅用作继承，就可以写成抽象类
除了不能实例化，不能创建对象之外，与普通类没有任何区别
继承层次很高，很高的祖先类，可以定义成抽象类，这种类太宽泛了，不适合用于创建对象

例如：动物，这个类太宽泛，被哺乳动物，爬行动物等类继承，当创建一个猴子对象时会用到哺乳动物类，而不是动物类，此时动物类就可以被设计为抽象类，太宽泛了

public abstract class Play {}

抽象方法

如果一个方法,没有具体的行为实现,仅有定义，被称为抽象类
1.需要用abstract关键字定义
2.没有方法体，没有具体的实现
3.不能定义为私有的，因为需要继承才能使用
4.子类必须对父类的抽象方法进行实现（重写），如果不进行实现，那子类也只能时抽象类
有抽象方法的类必须为抽象类，也只能定义于抽象类
一个抽象类可以没有抽象方法，但是有抽象方法的类必为抽象类

某些场景下，某些方法无法提供具体的行为实现，或者不适合提供时间，或者不想提供实现
太宽泛的行为，经常写在父类，被用于子类的重写

public abstract void run();

static关键字

static用于修饰类成员（属性或方法），隶属于类

static：静态的，非成员的
隶属于类的，而非隶属于对象的，属性和方法上使用
命名时全部大写命名
调用时直接用类名.属性名或方法名调用
public static int AGE = 22;

某些场景下，有一些成员（属性和方法）根本不需要创建对象，可以定义为static成员
例如：Math类，其中有很多静态方法，random()，调用时直接Math.random()

final关键字

final表示最终的，不可改变的
1.修饰变量：变量是不可变的最终的，将变成常量
2.修饰方法：表示此方法是不可变最终的，不可被重写
3.修饰类：表示此类是最终的不可变，不能被继承

接口

接口不是类，定义接口使用interface关键字
接口，可以约束某个类应该拥有哪些行为，并对其进行实现
如果多个类实现了此种接口，就可以提供相似的行为

1.接口中的方法必须是抽象方法（jdk8之后，有例外）
　接口中的方法默认是公有的public，抽象的abstract
　　　void run();省略了public，abstract
2.接口中的属性默认是公有的public，静态的static，最终且不可变的final
　　　String name = “张三”;省略了public，static，final

3.类可以实现接口，使用关键字implements来实现接口
　类实现了接口就须要对接口内的抽象方法提供实现
　一个类可以实现多个接口，需要将每个接口中的方法都提供实现
4.接口可以继承，且可以多继承，一个接口可以继承多个接口

多态:如果这个接口被多个类实现，那会有多态的表现
Runnable a = new Animal();//　Runnable是接口，Animal是类
a.run();//　run();是接口Runnable里的抽象方法，在类Animal中被实现
此时定义了接口类a，a调用的方法，只能是接口里的，被实现类所重写的方法，而不能调用实现类里其他的方法

八大基本数据类型：
byte short int long//整数
float double//小数
char//字符
boolean//1，0

对于每一种基本类型，都有对应的引用类型
byte：Byte
short：Short
int：Integer//-128-127的整数进行缓存，如果超过则new
long：Long
float：Float
double：Double
char：Character
boolean：Boolean

装箱操作：把一个基本数据类型赋值给引用类型对象，Java会自动转换
基本数据类型只是一个值，但引用类型是个对象，把基本数据类型打包打包封装到引用对象中———装箱
例：Integer a1 = Integer.valueOf(10);
简写为Integer a1 = 10;
把10这一int基本数据类型转换为Integer引用对象
机制是对-128-127的整数进行缓存，当要赋值的数在这个范围内，就直接在缓存中提取，如果超过这个范围则会new创建新的引用对象进行存储

拆箱操作：把一个引用类型对象赋值给基本数据类型，Java自动转换
a1.intValue();
int a2 = a1 + 10;//此时a1就自动执行了拆箱了操作a1.intValue();

XXX.parseXXX;//String转换为XXX基本数据类型
int a3 = Integer.parseInt(“123”);
注意这里转换成了基本数据类型int
XXX.valueOf();//String转换为XXX引用类型
Integer a4 = Integer.valueOf(“123”)
注意这里转换成了引用类型Integer
a3 == a4;//引用类型和基本数据类型判断，判断结果为true
进行了拆箱操作，a4转换成了基本类型
