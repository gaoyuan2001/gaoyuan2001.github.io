JavaSE基础语法

2023/1/12 学习 Java

六、异常：Exception

异常处理机制：Java语法，及异常类型

程序中出现非预期的情况，导致程序无法正常运行，数据出错，或者程序终止。

异常类型：
1.检查异常：强制捕获，编写的时候会标红，报异常，必须要进行捕获或抛出异常
2.运行时异常：RuntimeException，运行的时候报异常，并不要求强制执行

关键字：
try　catch　finally

throw　throws

throws：
抛出，写在方法参数列表后边，表示将异常抛给上层调用者，由上层调用者来处理，如果上层调用者依然无法处理，或者不合适处理，可以继续向上抛。
一个方法可以抛出多种类型的异常，使用逗号分隔

场景：当前方可以自行处理这个异常，就try catch，如果处理不了就throws，抛给上一级调用者，上一级调用者可以处理就try catch，如果还是处理不了，继续上抛，一层一层向上抛
构造函数也可以抛出异常

throw:
抛出。程序员手动抛出（触发）一个异常对象，意味着此方法可能出现异常。如果执行了抛出异常，此后的代码不会执行
手动自己写，在自己觉得可能会出现异常的地方，抛出异常，调用此方法的语句获取异常后，会对其做出处理（try catch），相当于提前预见异常，然后预处理，未雨绸缪

八、多线程与并行程序（并发执行程序）

单核CPU不存在并行程序，感觉不出来是切换时间很短
多核CPU，每个核心就相当于一个窗口，可以单独的执行程序，并发执行程序

进程：

process，独立的一个应用程序，QQ，word等等
进程与进程之间是独立的，依靠端口进行通讯，
一个应用程序至少由一个进程（QQ：聊天、自动升级），一个应用程序可能会有多个进程

线程：

线程是在进程之内的，一个进程可以包含多个线程（至少一个），多个线程共享进程的内存和数据
线程之间可以直接通讯，线程粒度比较小
线程执行没有任何规律，无法预测，无法控制
main（一个进程），main里只有一个线程

并发：
一个人，交替的做多个任务；即一个人执行一个任务，在这个任务执行期间，有一段时间是空闲的，他利用这段时间再去执行另一个任务，这样交替执行任务

并行：
多个人，同时做多个任务；总耗时取决于用时最长的任务

start()，只能说明进入就绪状态，启动线程
run()，才进入到执行状态
run()的数量和线程数有关系，每有一个run()就说明需要一个线程，如果run()的数量大于线程数，需要等到有线程执行完，才会被后面的run()使用

创建线程的几种方法（线程的创建是比较耗时的）：

1.创建子类继承Threat，在子类中重写run()方法

2.编写一个类，实现Runnable接口，创建Thread类的实例，以Runnable接口的对象作为Thread类构造函数的参数

3.使用jdk提供的线程池，来创建线程
Executor工具类
创建一个单线程的线程池
ExecutorService es = Executors.newSingleThreadExecutor();
创建一个固定线程数的线程池
ExecutorService es2 = Executors.newFixedThreadPool(10);
创建一个不限数量，但可以缓存的线程池
ExecutorService es3 = Executors.newCachedThreadPool();
创建固定线程数量的定时任务线程池

4.落后的实现定时任务的方式：Time类和TimerTask类集合，可以实现定时任务 

//创建类以创建内部类作为例子  

//第一种方法：创建子类继承Threat，在子类中重写run()方法  
// 匿名内部类，继承自Thread  
        // 创建一个线程对象  
        Thread t1 = new Thread() {  
            @Override  
            public void run() {  
                System.out.println("线程1");  
                for (int i = 1; i <= 100; i++) {  
                    System.out.println(i);  
                    Thread.yield();// 给CPU一个建议，让其尽量把时间分配给其他线程  
                }  
            }  
        };  
        
        t1.start();// 启动线程  


//第二种方法：编写一个类，实现Runnable接口，创建Thread类的实例，以Runnable接口的对象作为Thread类构造函数的参数          
        Runnable r = new Runnable() {
            @Override  
            public void run() {  
                for (int i = 1; i <= 100; i++) {  
                    System.out.println(i);  
                }  
            }  
        };  
        Thread t1 = new Thread(r);  
        Thread t2 = new Thread(r);  
        
        t1.start();//启动线程  
        t2.start();  


//第三种方法：线程池  

        // 1.创建一个单线程的线程池  
        ExecutorService es = Executors.newSingleThreadExecutor();  
        // 提交一个任务，一个线程  
        es.submit(new Runnable() {  
            @Override  
            public void run() {
                for (int i = 1; i <= 100; i++) {
                    System.out.println(i);
                }
            }
        });


        // 2.创建一个固定线程数的线程池
        ExecutorService es2 = Executors.newFixedThreadPool(10);// 10个线程的线程池
        for (int i = 0; i < 20; i++) {
            // 提交20个任务，10个任务完成后，空出来的线程继续执行后10个
            es2.submit(new Runnable() {
                @Override
                public void run() {
                    for (int i = 1; i <= 100; i++) {
                        System.out.println(i);
                    }
                }
            });
        }


        // 3.创建不限数量的线程池
        ExecutorService es3 = Executors.newCachedThreadPool();
        es3.submit(new Runnable() {
            @Override
            public void run() {
                for (int i = 1; i <= 100; i++) {
                    System.out.println(i);
                }
            }
        });


        // 4.创建一个指定线程数量的定时任务线程池
        ScheduledExecutorService ss1 = Executors.newScheduledThreadPool(10);
        ss1.schedule(new Runnable() {

            @Override
            public void run() {
                // TODO Auto-generated method stub
                System.out.println("bbbbbb");
            }
        }, 10, TimeUnit.SECONDS); // 每隔10秒执行一次
        
        ScheduledExecutorService ss2 = Executors.newScheduledThreadPool(10);
        ss2.scheduleAtFixedRate(new Runnable() {

            @Override
            public void run() {
                // TODO Auto-generated method stub
                System.out.println("aaaaaa");
            }
        }, 3, 5, TimeUnit.SECONDS); // 第3秒开始，之后每隔5秒执行一次
