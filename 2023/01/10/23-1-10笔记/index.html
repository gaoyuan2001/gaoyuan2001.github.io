六、异常：Exception

异常处理机制：Java语法，及异常类型

程序中出现非预期的情况，导致程序无法正常运行，数据出错，或者程序终止。

出现异常后，程序立即终止，异常位置之后的代码不再执行
由程序员去捕获异常，在捕获异常后，进行一定的补救或者挽回措施，同时保证程序可以继续运行
捕获私有前后顺序的。当出现异常时，只会进入一个分支

finally分支一定会执行，且优先级很高，比return优先级更高
最好不要再finally里写return

补充知识：return时，会提前将返回值锁定lock，以求返回前不被破坏

try　{ //程序

}　catch　(异常类型1)　{

}　catch　(异常类型2)　{

}　catch　(Exception　e)　{
　　e.printStackTrace();
}　finally　{//一定会执行

}

catch　(异常A　|　异常B){
}
此种语法异常A与异常B之间不能有继承关系

常见异常类型：
　　Exception：异常的父类
　　ArithmeticException：数学异常
　　RuntimeException：运行时异常
　　ArrayIndexOutOfBoundsException：数组下标越界
　　StringIndexOutOfBoundsException：字符串索引越界异常
　　IndexOutOfBoundsException：下标越界异常
　　NullPointerException：空指针异常

七、IO输入输出类型：InputStream，OutputStream

IO流：输入/输出流

I：Input，输入
O：Output，输出

按照流动的方向可分为：输入流和输出流
输入流：外界（控制台，文件，网络等）流入程序
输出流：程序流出外界（控制台，文件，网络等）

按照流动的单位可分为：字符流和字节流
字符：流动的字符。ABCD汉字数字等
字节：推荐，InputStream和OutputStream都是字节流

按照数据的介质可以分为：
1.文件流
2.数组流
3.网络
4.控制台
5.文件

I/O流涉及大量异常处理

InputStream：字节输入流，抽象类
OutStream：字节输出流，抽象类
BufferedInputStream：字节缓冲输入流，抽象类
BufferedOutputStream：字节缓冲输出流，抽象类

Reader：字符输入流，抽象
Writer：字符输出流，抽象
BufferedReader：字符缓冲输入流，抽象类
BufferedWriter：字符缓冲输出流，抽象类

路径一个使用/，或者使用\来进行分隔

字节流：InputStream，OutputStream
字符流：Reader，Writer

输入
创建一个文件对象，存储文件
创建一个文件字节/字符输入流对象，读取文件对象，被缓冲输入流引用
创建一个字符缓冲输入流对象，读取字符输入流对象，把文件放入缓冲输入流中
用字符缓冲输入流对象，调用方法读出流中的文件数据
关闭流对象close()

输出
创建一个文件对象，存储文件，用于存储输出的数据
把文件对象在硬盘中创建出来creatNewFile
创建一个文件字节/字符输出流对象，用于被缓冲流输出流引用
创建一个字符缓冲输出流对象，用于把输出数据放到流中
用缓冲输出流对象，调用方法读取输出数据，并传入缓冲流中
清空字符缓冲输出流，把流中数据写入到文件中
关闭流对象close()

ObjectInputStream：对象字节输入流
ObjectOutputStream：对象字节输出流
注意对象序列化问题
需要让对象的类实现一个接口Serializable
