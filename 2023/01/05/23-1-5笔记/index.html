三、String：

字符串对象是不可变的
引用指向的地址会变，但已经存在的字符串不会消失，也不会改变

和Integer类似，可以向上查看
String str1 = “hello”;// 是str2的简写，也是有缓存机制的
String str2 = String.valueOf(“hello”);// 启用缓存机制，和str1地址一样
String str3 = new String(“hello”);// 创建新的字符串对象和str1值一样，但是地址不一样

String str4 = “ja” + “va”;
“ja”创建一个对象，”va”创建一个对象，但”java”在缓存中肯定有，这门语言叫java，一共创建了两个对象
String str5 = “java” + “你好”;
“java”不会创建对象，”你好”会创建一个对象，”java你好”创建一个对象，一共创建了两个对象

但是这两个类存储的字符串对象内容可以变，且两者用法类似

StringBuffer(线程安全)，StringBuilder(线程不安全)

StringBuffer str6= new StringBuffer();
StringBuilder str7 = new StringBuilder();

历史遗留：

Calendar can = Calendar.getInstance();

int year = can.get(Calendar.Year);// 年

int year = can.get(Calendar.DAT_OF_WEEK);// 一个星期的第几天
int hours = can.get(calendar.HOUR_OF_DAY);// 小时

can.set(Calendar.HOUR_OF_DAT,20)// 设置小时

建议使用：

LocalDate，LocalTime，LocalDateTime
不能set，只能新建一个对象重新设置

LocalDate

本地日期类：年，月，日
LocalDate ld1 = LocalDate.now();// 静态方法
LocalDate ld2 = LocalDate.of(2001,1,24);// 引用ld2指向创建的新LocalDate对象(2001,01,24)
ld1.getYear();// 获取年
ld1.getMonthValue();// 获取月
ld1.getDayOfMonth();// 获取日

LocalDate ld3 = ld2.plusYears(10);// 加上10年,会生成一个新的对象，而原来的对象ld2不会改变
LocalDate ld4 = ld2.minusWeeks(3);// 减去3周，会生成一个新的对象，而原来的对象ld2不会改变

LocalTime

本地时间类：时间，时，分，秒，纳秒，1ms=1000000纳秒
LocalTime lt1 = LocalTime.now();// 静态方法
LocalTime lt2 = LocalTime.of(13,14,21);// 引用lt2指向创建的新LocalTime对象(13,14,21)

lt1.getHour();// 获取小时
lt1.getMinute();// 获取分钟
lt1.getSecond();// 获取秒

LocalDateTime

日期和时间都有

知识补充：

1.static:

静态的，固定值，固定的方法，属于类的
无需创建对象就可引用，通过类名就可以引用
静态成员只能访问静态成员，不能访问普通成员，因为访问普通成员需要创建对象才能访问，没有this，比如静态方法中只能访问静态属性
普通成员可以访问静态成员
静态成员可以访问构造函数，因为执行构造函数之前还没有对象

静态块：是在创建对象之前，初始化类的时候执行
有这个类时就会执行静态块内的代码
private static int count;
static {// 静态块
counter = 100;
}

非静态块：创建对象，执行构造函数，先执行第1句（super()或者this()，然后执行非静态块，再接着执行构造函数
{// 非静态块

}

2.变量的作用域与生命周期

作用域：变量生效的空间范围。从所在块，当前定义位置开始，一直到所在块结束，{块}
生命周期：变量生效的时间范围。从执行当前代码开始，一直到当块代码执行结束

块就是两个大括号之间的{　　块　　}

4.switch

分支结构
缺陷：仅能用于判断与目标值是否相等的情况
switch(a)　{
　　case　1:　{
　　　　break;
　　}
　　case　2:　{
　　　　break;
　　}
　　default:　{
　　}
}
或者
switch(b)　{
　　case　1　->　{
　　　　break;
　　}
　　case　2　->　{
　　　　break;
　　}
　　default　->　{
　　}
}
掉落机制：一旦进入一个分支，会继续执行以后的分支，除非加break跳出switch
default分支可以没有

5.内部类

Java语法特性
用的不多
在一个类的内部，定义一个类

成员内部类（非静态内部类）：也是外部类的一个成员
使用需要先创建外类的对象，然后用对象.创建成员内部类
内部类中，持有一个其外部类对象的引用，可以访问外部类的成员
如果成员内部类继承于另一个类，则间接实现了多继承
