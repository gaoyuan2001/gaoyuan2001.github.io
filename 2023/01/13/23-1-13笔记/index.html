八、多线程与并行程序（并发执行程序）

线程安全和同步

当多个线程同时（绝对同时或近乎同时）读写同一个共享的数据时，可能会出现数据不一致的情况，导致线程出现逻辑错误，称之为线程不安全的

线程安全：原子操作
单线程下不需考虑线程安全

ArrayList、HashSet、HashMap都不是线程安全的

Java web天生就是多线程的（例如可以多个用户同时操作），把多线程封装了

推荐书籍《Java世界》

多线程环境，如何编写线程安全的代码：

一、加锁：对代码加锁
一个井（共享数据），很多人（线程）来打水（访问数据），给井“加锁”，限制每次打水的人数，其他人等待

轻量级锁，重量级锁，Lock类
可重入锁，不可重入锁

重量级锁：Java语法原生支持的一种锁，同步锁。
synchronized，任意引用类型对象都可以作为锁
把代码放入到synchronized中

同步锁，可以解决线程安全问题，但是也会带来程序性能的急剧下降

同步块：
括号内是锁，锁可以是任意引用类型，但要保证各线程用的是同一个锁
synchronized(锁) {

}
只给会引发线程安全的语句加锁，对程序性能的影响降到最低 

private static final byte[] lock = new byte[0];    
synchronized(lock){//任意引用类型对象都可以作为锁，括号里的    
   list.add(i);     
}

同步方法：慎用！
以this为锁，保证各线程用的同一把锁，
将synchronized直接添加到方法返回值类型前面，性能急剧下降

如果synchronized用在static方法上，则以　类.class　为锁

二、尽量使用线程安全的类。StringBuffer，ConcurrentHashMap
1.把类包装成线程安全的类
使用Collections工具类包装成线程安全的。可以将List，Set，Map包装成线程安全的集合。
用Collections.synchronized———()
2.使用线程安全的类
ConcurrentHashMap，CopyOnWriteList，CopyOnWriteSet。

三、其他一些线程安全的技巧
比如：尽量避免数据共享，避免多线程等
